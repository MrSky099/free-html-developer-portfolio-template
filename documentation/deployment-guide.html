Deploy this Portfolio (Netlify & Vercel)

Clear, step-by-step guides for Free plans (static site, no server)

This project is a static site: index.html, style.css, script.js, and images/. No build step is required.

Quick Start (Most People)

  1. Put files at the project root (no subfolders for the main page). You should see: 
    index.html
style.css
script.js
images/
  2. Choose your platform: Netlify or Vercel. Both are great and free for static sites.
  3. Use Git import for automatic redeploys on every push. If you don’t use Git, use Netlify Drag & Drop.
  4. Custom domain (optional): Add your domain in the platform settings and follow the DNS instructions. HTTPS is automatic.

Prerequisites

  • GitHub, GitLab, or Bitbucket account (for Git-based deploys)
  • Netlify account (Free plan is enough)
  • Project files in a repository or a local folder

Vercel — Deploy via Git (recommended)

  1. 1) Push code to Git (GitHub, GitLab, or Bitbucket). Keep files at the repo root (see Quick Start).
  2. 2) Create a Vercel project
    • Sign in at Vercel.
    • Click “Add New…” → “Project” and choose Import Git Repository.
    • Authorize your Git provider and select your repository.
  3. 3) Configure project settings

    Build & Output Settings

    • Framework Preset: Other (no framework)
    • Build Command: (leave empty) (no build needed)
    • Output Directory: / (repo root with index.html)

    Environment variables (optional)

    If you later add APIs that need keys, set them in Settings → Environment Variables.

  4. 4) Deploy. Click “Deploy”. You’ll get a URL like https://your-project.vercel.app.
  5. 5) Continuous deployments. Every push to your default branch redeploys automatically. PRs create Preview Deployments.
If your files live in a subfolder (e.g., /site), set Output Directory to that folder.

Vercel — Custom Domain and HTTPS

  1. Open your project in Vercel → Settings → Domains.
  2. Add your custom domain and follow the DNS instructions (Vercel DNS or your registrar via CNAME/A records).
  3. HTTPS is automatic via Let’s Encrypt on the Free plan.

Vercel — Rollbacks and Previews

  • Preview Deployments: Every PR gets an isolated preview URL.
  • Promote to Production: Use the Promote action to make a previous deployment live.

Vercel — Free Plan Notes

  • Ideal for static sites like this portfolio.
  • Includes HTTPS, global edge network, preview deployments, and Git-based CI.
  • Serverless/Edge function limits, bandwidth, and other quotas apply. See Vercel’s current limits.
Limits can change. Confirm details in Vercel’s pricing and documentation.

Troubleshooting (Vercel)

  • 404 after deploy — Cause: Wrong Output Directory. Fix: Set Output Directory to the folder with index.html.
  • Missing styles/scripts — Cause: Incorrect paths. Fix: Use relative paths like ./style.css and ./script.js.
  • Images not loading — Cause: Wrong filename or path. Fix: Check images/ names and case-sensitivity.
  • Blank page — Cause: Moved files or broken HTML. Fix: Verify your index.html references and run locally before deploying.
  • Logs — Check the Deployments tab → select a deployment → Functions/Logs for details.

Netlify — Deploy via Git (recommended)

  1. 1) Push code to Git (GitHub, GitLab, or Bitbucket). Keep files at the repo root (see Quick Start).
  2. 2) Create a Netlify site
    • Sign in at Netlify.
    • Click “Add new site” → “Import an existing project”.
    • Choose your Git provider and authorize access.
    • Select your repository.
  3. 3) Configure basic settings

    Build settings

    • Build command: (leave empty) (no build needed)
    • Publish directory: / (repo root with index.html)

    Environment variables (optional)

    If you later add APIs that need keys, set them here.

  4. 4) Deploy. Click “Deploy site”. You’ll get a URL like https://your-site-name.netlify.app.
  5. 5) Continuous deployment. Every push to your default branch redeploys automatically. Pull requests create preview URLs.
If your files live in a subfolder (e.g., /site), set Publish directory to that folder.

Netlify — Drag & Drop (no Git)

  1. 1) Prepare: Zip your site so that index.html is at the root of the zip (not inside another folder).
  2. 2) Upload: Go to Netlify Drop and upload the zip or folder.
  3. 3) Done: You get a live URL instantly. To update, re-upload or connect a Git repo later.

Netlify — Custom Domain and HTTPS

  1. Open your site in the Netlify dashboard → Site configuration → Domain management.
  2. Add your custom domain and follow the DNS instructions (Netlify DNS or your registrar’s DNS via CNAME/A records).
  3. Enable automatic HTTPS with Let’s Encrypt (one click). It’s free on the Free plan.

Netlify — Rollbacks and Previews

  • Deploy Previews: Every PR can get an isolated preview URL.
  • Instant Rollbacks: Promote a previous deploy to production in one click if needed.

Netlify — Free Plan Notes

  • Perfect for static sites like this portfolio.
  • Includes HTTPS, global CDN, continuous deployment, and deploy previews.
  • Serverless functions, build minutes, and bandwidth have Free-tier limits. Check Netlify’s current quotas in their pricing docs.
Limits can change. Always confirm the latest details in Netlify’s pricing and documentation.

Troubleshooting (Netlify)

  • 404 after deploy: Ensure index.html exists at your publish directory root.
  • Missing styles/scripts: Paths in index.html must match your files (e.g., ./style.css, ./script.js).
  • Images not loading: Verify the images/ folder and filenames are correct and case-sensitive.
  • Wrong publish directory: Set it to the folder that contains index.html.

Need help? Open the site in Netlify and check Deploys → Logs for details.
